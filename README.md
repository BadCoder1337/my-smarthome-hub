# my-smarthome-hub

ESPHome-прошивка RF433-приёмника на ESP32 (ESP32-DevKitC). Заменяет Tasmota
Sonoff RF Bridge — принимает коды с 433 МГц пультов и отдаёт их в Home Assistant
как **именованные кнопки** (`binary_sensor.button_1..8`), одинаковые для всех
пультов (зеркально).

> Старый код (Node/TS хаб) сохранён в ветке [`legacy`](../../tree/legacy).

## Как это работает

Пульты — стандартный PT2262. Штатный `rc_switch` в ESPHome их не декодирует
(рассинхрон по sync), поэтому здесь свой external-компонент
[`components/rf_capture`](components/rf_capture): он слушает приёмник, сам
декодит кадр в 24-битный код, фильтрует по ID пульта и поднимает кнопку по
младшему ниблу кода (`8→1, C→2, 4→3, 9→4, 2→5, 5→6, 1→7, 3→8`).

## Использование в Home Assistant (ESPHome Dashboard)

1. Скопируй `secrets.yaml.example` → `/config/esphome/secrets.yaml`, подставь
   свои Wi-Fi, ключи и список пультов `rf_remotes`.
2. Добавь новое устройство в ESPHome Dashboard и вставь содержимое `rf-hub.yaml`.
   `external_components` тянет компонент прямо из этого репозитория с GitHub —
   ничего копировать не нужно.
3. Install → Wirelessly (или через USB при первой прошивке).

## Тайминги декодера (в блоке `rf_capture:`)

| параметр | по умолчанию | назначение |
|---|---|---|
| `bit_threshold` | `450us` | граница «длинный/короткий» импульс |
| `min_pulse` | `150us` | игнорировать импульсы короче |
| `max_pulse` | `1000us` | игнорировать импульсы длиннее |
| `hold_time` | `250ms` | длительность «нажатия» после последнего кадра |

## Добавить новый пульт

Нажми его кнопку рядом с платой, посмотри декодированный ID в логах ESPHome и
допиши 20-битный ID в `rf_remotes` (в `secrets.yaml`).

## Файлы

- `rf-hub.yaml` — конфиг прошивки.
- `components/rf_capture/` — кастомный компонент-декодер.
- `automations.yaml` — примеры HA-автоматизаций на именованные кнопки.
